Content Management System¶

Content management system using Pages and blocks in pages.

Requirements: lux.extensions.base, lux.extensions.api

The extension add two routers to the list of your application middleware.

Parameters¶

CREATE_PAGE_TEMPLATE¶

Default: Navigation-Center(div)

Function which return an Html element to which render the create new page form

NO_PAGE_TEMPLATE¶

Default: Navigation-Center(div)

Template Page for a response on a resource without own page

PAGE_EDIT_URL¶

Default: 'pages'

Base url for editing pages on the browser.

PAGE_TEMPLATES¶

Default: [Navigation-Content-Footer-Fluid(div), Navigation-Center(div)]

A list of Templates which can be used to render a page.

SITE_ID¶

Default: 1

The database id of the site

Usage¶

This adds a on_html_response callback for handling the html layout. The callback is implemented via the CmsResponse class.

Positioning and editing¶

When the configuration parageter PAGE_EDIT_URL is enabled, it is possible to edit html page layout by moving content around and adding new content.

  • A page specify a given template which in turns can have several (usually one or two) grid elements.
  • Each grid element contain one or more row elements.
  • Each row element contain one or more column elements.
  • Each column element contain one or more block elements, rendered vertically, one after the other.
  • Finally a block element contains one or more position elements which display a content.

The content is stored in the models.Content model.

Site Content¶

A special cms content type is the site content. Any Router with a get method implemented which can respond to a request for text/html content type, can be included as site content.

Site content can than be displayed anywhere on the web site thanks to the content management system positioning and editing functionalities.

To include a router, one must set the attribute cms_content to a title string used by the content management system in the drop-down selection of site contents. For example:

class SimpleContent(lux.Router):
    cms_content = 'A simple Hello'

    def get(self, request):
        return 'Hello!'

The views.EditPage.content_urls() method is responsible for collecting all valid site content to include in the drop-down selection element.

Routes & Hooks¶

Several Routers are used by this extension and all of them are also subclasses of the PageMixin which implements the retrieval of the page instance as well as the page template object.

The CmsRouter handles get requests on paths which are not served by any other router in your application. The EditPage is a CRUD router which serves the page editing urls.

Page Components¶

The lux.extensions.cms.grid module contains several components for creating PageTemplate to use with the content management system.

PageTemplate¶

class lux.extensions.cms.grid.PageTemplate(*children, **params)[source]¶

The main Template of the content management system extension.

The template renders the inner part of the HTML body tag. A page template is created by including the page components during initialisation, for example:

from lux.extensions.cms.grid import PageTemplate

head_body_foot = PageTemplate(
    Row()
    Grid(CmsContext('content')),
    Grid(CmsContext('footer', all_pages=True)))

Grid¶

class lux.extensions.cms.grid.Grid(*children, **parameters)[source]¶

A container of Row or CmsContext templates.

Note that a Grid template can be used independently from the content management system.

Parameters:fixed – optional boolean flag to indicate if the grid is fixed (html class grid fixed) or fluid (html class grid fluid). If not specified the grid is considered fluid (it changes width when the browser window changes width).

Row¶

class lux.extensions.cms.grid.Row(*children, **params)[source]¶

A Row is a container of Column elements. It must contain at least one Column.

Note that a Row template can be used indipendently from the content management system.

Parameters:column – Optional parameter which set the column attribute.
column¶

It can be either 12 or 24 and it indicates the number of column spans available.

Default: 24

Column¶

class lux.extensions.cms.grid.Column(*children, **params)[source]¶

A column can have one or more Block.

CmsContext¶

class lux.extensions.cms.grid.CmsContext(key, *children, **params)[source]¶

A specialised Context for rendering the dynamic content in a PageTemplate.

The dynamic content is retrieved from data in a backend database.

Parameters:all_pages – optional boolean to pass during initialisation, it sets the all_pages attribute
all_pages[source]¶

When True, this context is used by all pages.
