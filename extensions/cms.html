Content Management System¶

Content management system using Pages and blocks in pages.

Requirements: lux.extensions.base, lux.extensions.api

Features¶

  • Programmatically specify one or several templates for HTML pages
  • A page template has one or more grid elements
  • Each grid element contain one or more row elements
  • Each row element contain one or more column elements
  • Each column element contain one or more block elements, rendered vertically, one after the other
  • Finally a block element contains one or more position elements which display a content
  • The content is stored in the models.Content model
  • Inline editing of pages and content within a page when the PAGE_EDIT_URL parameter is enabled

Contents¶

Contents within a page are of three types:

  • A simple json object stored within a block in the Page.content object. These objects tend to be simple and are implemented in javascript only.
  • A content defined by Router with cms_content attribute which evaluates to True.

Site Content¶

A special cms content type is the site content. Any Router with a get method implemented which can respond to a request for text/html content type, can be included as site content.

Site content can than be displayed anywhere on the web site thanks to the content management system positioning and editing functionalities.

To include a router, one must set the attribute cms_content to a title string used by the content management system in the drop-down selection of site contents. For example:

class SimpleContent(lux.Router):
    cms_content = 'A simple Hello'

    def get(self, request):
        return 'Hello!'

The views.EditPage.content_urls() method is responsible for collecting all valid site content to include in the drop-down selection element.

Template Page for a response on a resource without own page

Default: 'pages'

Base url for editing pages on the browser.

Default: '[Navigation-Content-Footer-Fluid(div), Navigation-Content-Footer-Fixed(div), Navigation-Center(div)]'

A list of Templates which can be used to render a page.

SITE_ID¶

Default: 'None'

The database id of the site

API¶

This adds a on_html_response callback for handling the html layout. The callback is implemented via the CmsResponse class.

Routes & Hooks¶

Several Routers are used by this extension and all of them are also subclasses of the PageMixin which implements the retrieval of the page instance as well as the page template object.

The CmsRouter handles get requests on paths which are not served by any other router in your application. The EditPage is a CRUD router which serves the page editing urls.

Cms Router¶

class lux.extensions.cms.views.CmsRouter(*args, **kwargs)[source]¶

A router for rendering a CMS page on the web.

This should be the last Router in the list of middleware of your application. It picks up any url which has not been served by any of the previous routers.

The get method is the only method available.
