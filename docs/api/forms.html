Forms API¶

A form library crucial not only for browser driven applications, but also for validating remote procedure calls (RPC) and command line inputs. The main class in this module is Form. It encapsulates a sequence of form Field and a collection of validation rules that must be fulfilled in order for the form to be accepted. Form classes are created as subclasses of Form and make use of a declarative style similar to django forms.

A Form is rendered into html via a Layout class. For example, this is a form for submitting an issue:

from lux import forms, html_factory

class IssueForm(forms.Form):
    name = forms.CharField()
    description = forms.CharField(required=False,
                                  widget=html_factory('textarea'))
    labels = forms.CharField(required=False)

    layout = forms.Layout(style='horizontal')

As you can see, no much has been specified in this declaration. Sensible defaults are used.

Form¶

class lux.forms.form.Form(request=None, data=None, files=None, initial=None, prefix=None, manager=None, instance=None)[source]¶

Base class for forms.

It can be used for browser based applications as well as remote procedure calls validation. If data is not None, this Form bind itself to the data, otherwise it remains unbounded.

Parameters:
  • data – dictionary type object containing data to validate. It sets the rawdata attribute.
  • files – dictionary type object containing files to upload.
  • initial – set the initial attribute.
  • prefix – set the prefix attribute.
  • manager – An optional manager for a database model.
  • instance – An optional instance of a model class.
  • request – Optional client request.
is_bound¶

If True the Form has rawdata which can be validated.

initial¶

Dictionary of initial values for fields. The values in initial are only used when is_bound is False.

rawdata¶

The input data, if available this Form is bound and can be validated via the is_valid() method.

prefix¶

String to use as prefix for field names.

Default: ''.

request¶

An instance of a Http request class stored for convenience. The Form itself does not use it, however user’s implementations may want to access it. In custom validation functions for example. Default None.

forms¶

A list of Form classes. If available, the forms are used to create sub-forms which are included in the current form.

Default: [].

form_sets¶

A disctionary of FormSet instances. If available, form-sets are used to create a given number of sub-forms which are included in the current form.

Default: {}.

instance¶

An optional instance of a model.

Default: None.

layout¶

The Html layout for this Form. It is a descriptor for rendering a Form instance into HTML:

form = MyForm(...)
html = form.layout(method='post', action='.')

Field¶

class lux.forms.fields.Field(required=None, default=None, initial=None, validation_error=None, help_text=None, label=None, widget=None, widget_attrs=None, disabled=None, attrname=None, wrong_value_message=None, **kwargs)[source]¶

Base class for all Form fields. Field are specified as attribute of a form, for example:

from lux import forms

class MyForm(forms.Form):
    name = forms.CharField()
    age = forms.IntegerField()
Parameters:
  • required – set the required attribute.
  • default – set the default attribute.
  • initial – set the initial attribute.
  • widget – Optional callable to override the widget attribute. If supplied it must accept this field as first parameter. Check the field_widget() factory for an example signature.
  • wrong_value_message – callable which receive the field and the field value when to produce a message when the value did not validate.
required¶

boolean specifying if the field is required or not. If a field is required and it is not available or empty it will fail validation.

Default: True.

default¶

Default value for this field. It can be a callable accepting a BoundField instance for the field as only parameter.

Default: None.

initial¶

Initial value for field. If Provided, the field will display the value when rendering the form without bound data. It can be a callable which receive a Form instance as argument.

Default: None.

See also

Inital is used by Form and by HtmlForm instances to render an unbounded form. The Form.initials() method return a dictionary of initial values for fields providing one.

widget¶

The djpcms.html.WidgetMaker for this field.

Default: None.

widget_attrs¶

dictionary of widget attributes used for setting the widget html attributes. For example:

widget_attrs = {'title':'my title'}

It can also be a callable which accept a BoundField as the only parameter.

Default: None.

CharField¶

class lux.forms.fields.CharField(required=None, default=None, initial=None, validation_error=None, help_text=None, label=None, widget=None, widget_attrs=None, disabled=None, attrname=None, wrong_value_message=None, **kwargs)[source]¶

A text Field which introduces three optional parameter (attribute):

max_length¶

If provided, the text length will be validated accordingly.

Default None.

char_transform¶

One of None, u for upper and l for lower. If provided converts text to upper or lower.

Default None.

toslug¶

If provided it will be used to create a slug text which can be used as URI without the need to escape. For example, if toslug is set to “_”, than:

bla foo; bee

becomes:

bla_foo_bee

Default None
