Sitemap and Navigation¶

An extension for managing navigation on a web site. The extension does not add any middleware to the lux WSGI callable, instead it adds an handler to the on_html_document event. In other words, the sitemap extension operates on text/html responses only.

Usage¶

When the WsgiRequest.html_document attribute is accessed for the first time, the extension, via the on_html_document() event, adds a NavigationInfo object to the request.cache dictionary. This object can be accessed via request.cache.html_navigation if needed.

The html_navigation can be used to render a navigation toolbar via the Navigation template.:

>>> from lux.extensions.sitemap import Navigation
>>> toolbar = Navigation()
>>> html = toolbar.render()

This extension should be positioned, within the EXTENSIONS list, before extensions which need to access the html_navigation instance in the request.cache dictionary. Since it does not add any wsgi middleware, it is safe to add it as the first extension.

Adding entries¶

The NavigationInfo works by looping over all wsgi middleware which is a instance of Router and check if the navigation_visit method is available.

For example this router implement a ‘doc’ link:

class SomeRouter(lux.Router):

    def navigation_visit(self, request, nav):
        ...

For example, in the cms extensions, the EditPage router adds links to edit or exit-editing of pages.

Parameters¶

NAVIGATION_BRAND¶

Default: ''

Html string for the brand in the toolbar

NAVIGATION_BRAND_LINK¶

Default: '/'

The href value of the anchor which display the brand in the navigation toolbar

NAVIGATION_CLASSES¶

Default: 'nav nav-list standard'

NAVIGATION_LEVELS¶

Default: '1'

NAVIGATION_TEMPLATE¶

Default: "('brand', 'main', 'secondary', 'user')"

Layout of the Navigation Toolbar
