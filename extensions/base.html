Base¶

The base lux extension, it provides several middleware utilities. If used, it should be the first extension in your EXTENSIONS list (excluding extensions which don’t provide any wsgi middleware such as lux.extensions.sitemap for example).

Parameters¶

CLEAN_URL¶

Default: True

When True, requests on url with consecutive slashes are converted to valid url and redirected.

CSS¶

Default: {}

Dictionary of css locations.

FAVICON¶

Default: None

Adds wsgi middleware to handle favicon url /favicon.ico``served from ``MEDIA_URL/FAVICON

GZIP_MIN_LENGTH¶

Default: 200

If a positive integer, a response middleware is added so that it encodes the response via the gzip algorithm.

HTML_META¶

Default: [{'content': 'IE=edge', 'http-equiv': 'X-UA-Compatible'}, {'name': 'viewport', 'content': 'width=device-width, initial-scale=1'}]

List of default meta elements to add to the html headelement

MEDIA_URL¶

Default: 'media'

the base url for static files

MINIFIED_MEDIA¶

Default: True

Use minified media files. All media files will replace their extensions with .min.ext. For example, javascript links *.js become *.min.js

SERVE_STATIC_FILES¶

Default: False

if True add middleware to serve static files.

USE_ETAGS¶

Default: False

Media Handling¶

When the SERVE_STATIC_FILES parameter is set to True, this extension adds middleware for serving static files from MEDIA_URL. In addition, a FAVICON location can also be specified.
