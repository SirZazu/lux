AngularJS Integration¶

This extension does not provide any middleware but it is required when using lux.js javascript module and provides the link between AngularJS and Python.

Required extensions: lux.extensions.ui

Usage¶

Include lux.extensions.angular into the EXTENSIONS list in your config file:

EXTENSIONS = [
    ...
    'lux.extensions.ui',
    'lux.extensions.angular'
    ...
    ]

 ANGULAR_UI_ROUTER = True

There are two different ways to serve html. The default way is to serve without the use of angular ui-router, in which case urls load the whole html5 document each time they are accessed.

UI-Router¶

When switching ANGULAR_UI_ROUTER to True, each page served by an angular Router is managed, in javascript, by the ui-router as a single page application.

class lux.extensions.angular.Router(*args, **kwargs)[source]¶

A Router for Angular navigation.

html_body_template = None¶

The template for the body part of the Html5 document. In most cases it is as simple as:

$html_main

where the $html_main is replaced by the string created by the build_main() method.

ngmodules = None¶

Optional list of angular modules to include

uirouter = 'lux.ui.router'¶

Override the ANGULAR_UI_ROUTER setting for this Router. If set to False the ui-router won’t be used and the whole html5 document is loaded when accessed.

get(request)[source]¶

This is the only http method implemented.

If this Router is operating in ui mode, this method performs the following tasks:

  • Collets children routers operating in ui mode and build the sitemap used by angular ui-router.

  • The $html_main content, rendered by the build_main() method is wrapped by a div element:

    <div id="seo-view" data-ui-main="main" class="hidden">
    $html_main
</div>
